## 1. 공지 에디터 이미지 — S3 + CloudFront 저장

에디터 이미지를 본문 HTML 에 base64 로 박던 방식을 **S3 업로드 + CDN URL** 방식으로 교체했다. 상세 설계는 별도 문서 [`notice-image-storage.md`](./notice-image-storage.md) 참고.

- 방식: 브라우저 → 우리 API(멀티파트) → S3 (백엔드 프록시, S3 CORS 불필요)
- 저장 키: `saas-admin/notices/images/{yyyy}/{MM}/{UUID}.{ext}` (food-biz 버킷 공유, 접두어로 분리)
- 업로드 실패/저장소 비활성 시 base64 폴백 → 에디터는 항상 동작

**백엔드**
- `pom.xml` — `software.amazon.awssdk:s3:2.25.16`
- `application.yml` — `storage.s3.*` (bucket `foodbiz-uploads`, CDN `d2ziky4ycezd5d.cloudfront.net`, `key-prefix: saas-admin`), 멀티파트 5MB/8MB
- `file/S3Config.java`, `file/FileUploadService.java`, `file/FileUploadController.java`(`POST /api/platform-admin/files/images`)
- `ErrorCode` — `FILE_EMPTY/FILE_TYPE_NOT_ALLOWED/FILE_TOO_LARGE/FILE_STORAGE_DISABLED/FILE_UPLOAD_FAILED`
- `GlobalExceptionHandler` — `MaxUploadSizeExceededException` → 413

**프런트**
- `api/client.js` — `fileApi.uploadImage()`
- `NoticesPage.jsx` `RichEditor.insertImageFile()` — base64 미리보기 즉시 삽입 후 업로드되면 `<img src>` 를 CDN URL 로 교체

**프런트**

## 6. 공지 권한 정책

- **댓글 삭제**: 작성자 **본인만** (슈퍼여도 남의 댓글 불가) — `canDeleteComment`
- **공지 수정·삭제**: **작성자 또는 직책(job_title)='대표'** — 슈퍼여도 직책이 대표가 아니면 남의 공지 불가 (`canEditNotice`/`isRepresentative`)
- 프런트는 `editable`/`deletable` 플래그로 버튼 노출 제어, 서버가 403 으로 강제

## 7. 슈퍼 해제 · 권한관리 접근 방식 변경

- **전 계정 `is_super=N`** (260001, 260008). 260001 은 조직 1·직책 대표이고 `DEPT_TITLE "1:대표"` 에 전 메뉴가 저장돼 있어 권한 테이블로 전 메뉴 접근 유지.
- **권한관리 화면 접근을 "슈퍼만" → "권한관리 메뉴(/permissions) 접근 권한 보유자"로 변경**
  - `PermissionService.hasMenuAccess(admin, url)` 추가
  - `PermissionController` / `CalendarController` 의 권한 API 가 이 판정을 사용 (`requireSuper` 제거)
  - 프런트 `PermissionsPage` 의 `!isSuper` 차단 블록 제거 (접근 통제는 라우트 가드 + 메뉴 권한)

## 8. 조직도 — 부서(조직) 드래그 이동

기존엔 조직도에서 **사원**만 드래그 가능했다. **부서(조직 노드)** 도 드래그로 옮길 수 있게 했다.

- `AdminsPage.jsx`
  - `dragOrgId` 상태, `moveOrg(targetOrgId)` → `orgApi.move(orgId, { parentId, position: 9999 })`
  - `onNodeDrop` — 사원 드래그면 배치, 조직 드래그면 이동으로 분기
  - `isDescendantOrg` — 자기 자신/하위로 이동 금지(서버도 `ORG_CYCLE`)
  - 부서를 다른 부서로 끌면 하위로, 조직도 빈 곳으로 끌면 최상위로
- `OrgTreeNode` — `.org-head` `draggable`, 드롭 시 `stopPropagation`
- 백엔드 move API 는 기존 것 사용(변경 없음)

## 9. 조직 배치 시 부서 라벨 동기화 (버그 수정)

**증상**: 조직도에서 사원을 다른 부서로 옮기면 트리는 바뀌는데 **관리자 상세보기의 '부서'는 옛날 값** 그대로.

**원인**: `assignMember` 가 `orgId` 만 바꾸고 `department`(상세보기 라벨)를 안 바꿈. (실제로 260007 이 org 3 인데 department="플랫폼본부" 로 stale)

**수정**: 배치 시 그 조직의 부서코드(`org_code`)에 대응하는 **DEPARTMENT 공통코드 이름**으로 `department` 동기화. 미배치(null)면 부서도 비움.
- `CommonCodeRepository.findByGroupGroupCodeAndCode`
- `AdminAccount.syncDepartment`
- `OrgService.assignMember` + `departmentNameOf`

검증: org5→"인프라팀", org3→"경영지원본부", null→null 로 정상 동기화(260007 stale 데이터도 교정됨).
